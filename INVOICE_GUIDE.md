# دليل إنشاء الفواتير التلقائي

يهدف هذا الدليل إلى توضيح كيفية إنشاء فواتير جديدة تلقائيًا باستخدام السكربت المخصص `generate_invoice.py`، بالإضافة إلى التعليمات اليدوية في حال الحاجة.

## 1. استخدام سكربت `generate_invoice.py` (الطريقة الموصى بها)

هذا السكربت يبسط عملية إنشاء الفواتير الجديدة عن طريق طلب المعلومات الأساسية منك ثم يقوم بتوليد ملف `index.html` في المسار الصحيح وتهيئته تلقائيًا.

### خطوات الاستخدام:

1.  **تأكد من وجود السكربت:** تأكد من أن ملف `generate_invoice.py` موجود في الجذر الرئيسي للمستودع (`/home/ubuntu/gs_manual/`).
2.  **تشغيل السكربت:** افتح الطرفية (Terminal) وانتقل إلى مجلد المستودع، ثم قم بتشغيل السكربت:
    ```bash
    python3 generate_invoice.py
    ```
3.  **إدخال البيانات:** سيطلب منك السكربت إدخال المعلومات التالية:
    *   `INVOICE_NUMBER`: رقم الفاتورة (مثال: 002400)
    *   `RANDOM_CODE`: كود عشوائي مكون من 6 أحرف (بدون O و I وبدون أرقام، مثال: ABCDEF)
    *   `CLIENT_NAME`: اسم العميل (مثال: شركة العميل المحدودة)
    *   `DATE`: تاريخ الفاتورة (مثال: ٥-٣-٢٠٢٦)
    *   `DURATION`: مدة التنفيذ (مثال: 20 يوم)
    *   `TOTAL_PRICE`: السعر الإجمالي (مثال: 2500 ريال سعودي)

4.  **مراجعة الملف:** بعد إدخال البيانات، سيقوم السكربت بإنشاء المجلد والملف `index.html` في المسار `/v/{INVOICE_NUMBER}{RANDOM_CODE}/index.html`.
5.  **رفع التغييرات إلى GitHub:**
    *   انتقل إلى مجلد المستودع:
        ```bash
        cd /home/ubuntu/gs_manual
        ```
    *   أضف الملفات الجديدة:
        ```bash
        git add .
        ```
    *   نفّذ commit برسالة مناسبة (مثال: `Add invoice {INVOICE_NUMBER}{RANDOM_CODE}`):
        ```bash
        git commit -m "Add invoice 002400ABCDEF"
        ```
    *   ارفع التغييرات إلى الفرع الرئيسي `main`:
        ```bash
        git push origin main
        ```
6.  **التحقق من الرابط:** انتظر بضع دقائق حتى تكتمل عملية بناء GitHub Pages، ثم تحقق من الرابط التالي:
    ```
    https://moalarbi.github.io/gs/v/{INVOICE_NUMBER}{RANDOM_CODE}/
    ```
    (استبدل `{INVOICE_NUMBER}` و `{RANDOM_CODE}` بالقيم التي أدخلتها).

## 2. إنشاء الفاتورة يدويًا (في حال عدم توفر السكربت أو الحاجة لتعديلات متقدمة)

إذا كنت تفضل إنشاء الفاتورة يدويًا أو تحتاج إلى تعديلات معقدة لا يدعمها السكربت، اتبع الخطوات التالية:

1.  **إنشاء المجلد:** أنشئ مجلدًا جديدًا بالصيغة `/v/{INVOICE_NUMBER}{RANDOM_CODE}/` داخل المستودع. تأكد من أن المسار يبدأ بـ `/v/` مباشرة.
    *   مثال: `/home/ubuntu/gs_manual/v/002400ABCDEF/`
2.  **إنشاء ملف `index.html`:** داخل المجلد الذي أنشأته، قم بإنشاء ملف باسم `index.html`.
3.  **نسخ محتوى القالب:** انسخ محتوى قالب الفاتورة (الموجود في `template/invoice_template.html` أو أي فاتورة سابقة تم إنشاؤها) والصقه في ملف `index.html` الجديد.
4.  **تعديل البيانات:** قم بتعديل البيانات التالية داخل ملف `index.html`:
    *   `id="invoice-number"`: رقم الفاتورة.
    *   `تاريخ الفاتورة`: التاريخ.
    *   `company-name`: اسم العميل.
    *   `مدة التنفيذ`: المدة.
    *   `amount`: السعر الإجمالي.
    *   تأكد من إزالة أي `item-count` (أسعار فردية) من بنود الاتفاقية.
    *   تأكد من أن رابط الواتساب في الفوتر صحيح: `https://wa.me/966537311886`.
    *   تأكد من أن رابط الموقع في الفوتر صحيح: `https://golf-stusio.umso.co/`.
5.  **رفع التغييرات إلى GitHub:** اتبع نفس الخطوات المذكورة في النقطة 5 من قسم "استخدام سكربت `generate_invoice.py`" لرفع التغييرات.
6.  **التحقق من الرابط:** اتبع نفس الخطوات المذكورة في النقطة 6 من قسم "استخدام سكربت `generate_invoice.py`" للتحقق من الرابط.

## ملاحظات هامة:

*   **التوكن:** تأكد من أن توكن GitHub الخاص بك مفعل ولديه الصلاحيات اللازمة للوصول إلى المستودع (`repo` scope).
*   **GitHub Pages:** قد يستغرق ظهور التغييرات على GitHub Pages بضع دقائق بعد عملية الـ push.
*   **هيكلة المسار:** من الضروري الالتزام بهيكلة المسار `/v/{INVOICE_NUMBER}{RANDOM_CODE}/index.html` لضمان عمل GitHub Pages بشكل صحيح.
